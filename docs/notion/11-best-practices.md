# 추가 팁 및 모범 사례

## 개요

이 문서는 노션 데이터베이스를 효과적으로 활용하기 위한 추가 팁과 모범 사례를 제공합니다.

## 1. 템플릿 활용

### 템플릿 생성 방법

각 데이터베이스에 템플릿을 생성하여 일관된 데이터 입력을 보장합니다.

#### 예시: 평가 템플릿

1. 평가 데이터베이스 열기
2. "템플릿" 버튼 클릭
3. 템플릿 페이지 생성
4. 기본값 설정:
   - 상태: Draft
   - 평가일시: 비어있음
5. 가이드 작성:
   - 능력단위 선택 방법
   - 학생 선택 방법
   - 교사 선택 방법
   - 점수 입력 방법

#### 템플릿 활용 예시

**평가 템플릿**:
- 능력단위 선택
- 학생 선택
- 교사 선택
- 기본 상태: Draft

**과제물 제출 템플릿**:
- 평가일정 선택
- 제출 유형 선택
- 파일 업로드 가이드

---

## 2. 뷰 활용

### 뷰 생성 전략

자주 사용하는 필터와 정렬을 뷰로 저장합니다.

#### 예시 뷰

**평가 데이터베이스**:
- "전체 평가" (기본 뷰)
- "진행 중인 평가" (상태 = In Progress)
- "이번 달 평가" (평가일시 필터)
- "미완료 평가" (상태 = Draft)
- "완료된 평가" (상태 = Confirmed)
- "높은 점수 평가" (총점 >= 90)

**과제물 제출 데이터베이스**:
- "전체 제출물"
- "미제출 과제물" (제출일시가 비어있음)
- "최근 제출물" (제출일시 정렬)
- "이미지 제출물" (제출 유형 = Image)

**평가일정 데이터베이스**:
- "전체 일정"
- "진행 중인 일정" (상태 = In Progress)
- "예정된 일정" (상태 = Scheduled)
- "이번 주 일정" (시작일시 필터)

## 3. 페이지 구조

### 권장 워크스페이스 구조

노션 워크스페이스를 다음과 같이 구성합니다:

```
📁 NCS 평가 시스템
  📄 대시보드
    - 관리자 대시보드
    - 교사 대시보드
    - 학생 대시보드
  📁 훈련과정 관리
    - [훈련과정 데이터베이스]
    - [능력단위 데이터베이스]
    - [능력단위요소 데이터베이스]
    - [수행준거 데이터베이스]
  📁 평가 관리
    - [평가일정 데이터베이스]
    - [평가 데이터베이스]
    - [평가 점수 상세 데이터베이스]
  📁 과제물 관리
    - [과제물 제출 데이터베이스]
  📁 사용자 관리
    - [프로필 데이터베이스]
    - [사용자 설정 데이터베이스]
  📁 서명 관리
    - [서명 데이터베이스]
  📁 관계 테이블
    - [훈련과정-교사 데이터베이스]
    - [훈련과정-학생 데이터베이스]
```

## 4. 권한 관리

### 역할별 권한 설정

노션 워크스페이스 권한을 역할별로 설정합니다.

#### 관리자

- **권한**: 전체 접근 권한
- **접근 가능**: 모든 데이터베이스 및 페이지
- **설정 방법**:
  1. 페이지/데이터베이스 공유 설정
  2. 권한: "편집 가능" 또는 "전체 액세스"

#### 교사

- **권한**: 평가, 과제물, 평가일정 접근
- **접근 가능**:
  - 평가 데이터베이스
  - 과제물 제출 데이터베이스
  - 평가일정 데이터베이스
  - 자신의 프로필
- **설정 방법**:
  1. 해당 데이터베이스 공유 설정
  2. 권한: "편집 가능"

#### 학생

- **권한**: 자신의 평가, 과제물만 접근
- **접근 가능**:
  - 자신의 평가 (필터: 학생 = [현재 사용자])
  - 자신의 과제물 (필터: 학생 = [현재 사용자])
  - 자신의 프로필
- **설정 방법**:
  1. 데이터베이스 공유 설정
  2. 권한: "뷰만 가능"
  3. 필터 뷰 생성 (학생 = [현재 사용자])

## 5. 자동화 (노션 API 활용)

### 노션 API 개요

노션 API를 사용하여 자동화를 구현할 수 있습니다.

#### 활용 예시

**Supabase에서 데이터 변경 시 노션 동기화**:
- Supabase 웹훅 설정
- 노션 API로 데이터 업데이트
- 실시간 동기화

**평가 완료 시 알림 전송**:
- 평가 상태가 "Confirmed"로 변경 시
- 노션 API로 알림 전송
- 관련자에게 이메일/푸시 알림

**주간 리포트 자동 생성**:
- 주간 평가 통계 수집
- 노션 API로 리포트 페이지 생성
- 자동 공유

### API 설정 방법

1. 노션 통합 생성
   - 노션 설정 → 연결 → 새 통합
   - 통합 이름 입력
   - 권한 설정
   - 토큰 발급

2. API 사용
   - 노션 API 문서 참고
   - Python, JavaScript 등으로 구현

## 6. 성능 최적화

### 데이터베이스 최적화

#### 1. 불필요한 속성 숨김

- 자주 사용하지 않는 속성은 숨김
- 필요한 속성만 표시

#### 2. 필터 활용

- 대량의 데이터는 필터로 제한
- 필요한 데이터만 표시

#### 3. 롤업 속성 최소화

- 필요한 롤업만 추가
- 과도한 롤업은 성능 저하

#### 4. 수식 속성 최적화

- 간단한 수식 사용
- 복잡한 수식은 피하기

---

## 7. 데이터 백업 및 복구

### 자동 백업

- 노션은 클라우드 기반으로 자동 백업
- 별도 백업 설정 불필요

### 버전 히스토리

1. 페이지 우측 상단 "..." 메뉴
2. "히스토리" 선택
3. 이전 버전 확인 및 복구

### 페이지 복구

- 삭제된 페이지는 30일간 복구 가능
- 휴지통에서 복구

## 11. 문제 해결

### 일반적인 문제

#### 문제 1: 데이터베이스가 느림

**해결**:
- 필터 사용
- 불필요한 속성 숨김
- 페이지 크기 제한

#### 문제 2: 관계가 작동하지 않음

**해결**:
- 관계 속성 확인
- 레코드 이름 확인
- 관계 재설정

#### 문제 3: 롤업 속성이 0으로 표시됨

**해결**:
- 관계 확인
- 롤업 설정 확인
- 데이터 연결 확인
