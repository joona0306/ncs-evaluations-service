# Supabase 단점 보완 방안

## 개요

이 문서에서는 Supabase의 제약사항을 노션의 강력한 기능으로 보완하는 방법을 설명합니다.

## 1. 시각화 및 대시보드

### Supabase 단점

- 데이터 시각화가 제한적
- 대시보드 구성이 복잡함
- 다양한 뷰 제공 부족

### 노션 보완

- 각 데이터베이스를 다양한 뷰(테이블, 보드, 타임라인, 갤러리)로 표시
- 필터와 정렬을 통한 동적 뷰 생성
- 템플릿을 통한 대시보드 페이지 구성

### 구현 방법

1. **대시보드 페이지** 생성
2. 각 데이터베이스를 임베드하여 배치
3. 필터와 정렬 설정으로 원하는 데이터만 표시

**자세한 내용**: [대시보드 페이지 제작 가이드](./10-dashboard-guide.md)

---

## 2. 협업 및 댓글

### Supabase 단점

- 실시간 협업 기능 부족
- 댓글 기능 없음
- 알림 시스템 제한적

### 노션 보완

- 페이지/데이터베이스 레코드에 댓글 추가
- @멘션을 통한 알림
- 페이지 공유 및 권한 관리

### 구현 방법

#### 평가 레코드에 댓글 추가

1. 평가 레코드 열기
2. 댓글 아이콘 클릭
3. 댓글 작성 및 @멘션으로 관련자 알림

#### 권한 관리

1. 페이지/데이터베이스 공유 설정
2. 역할별 권한 설정:
   - **관리자**: 전체 접근 권한
   - **교사**: 평가, 과제물, 평가일정 접근
   - **학생**: 자신의 평가, 과제물만 접근

---

## 3. 문서화 및 설명

### Supabase 단점

- 메타데이터나 설명 추가가 제한적
- 각 레코드에 상세 정보 추가 어려움
- 템플릿 기능 없음

### 노션 보완

- 데이터베이스 속성에 설명 추가
- 각 레코드를 페이지로 열어 상세 설명 작성
- 템플릿을 통한 일관된 데이터 입력

### 구현 방법

#### 1. 데이터베이스 속성 설명 추가

1. 데이터베이스 속성 편집
2. 속성 설명 추가
3. 저장

#### 2. 템플릿 페이지 생성

1. 데이터베이스에서 "템플릿" 버튼 클릭
2. 템플릿 페이지 생성
3. 기본값 설정 및 가이드 작성

#### 3. 각 레코드 페이지에 추가 정보 작성

1. 레코드 열기
2. 페이지에 추가 정보 작성
3. 이미지, 파일 등 첨부

---

## 4. 검색 및 필터링

### Supabase 단점

- 복잡한 쿼리 작성 필요
- 필터 UI가 제한적
- 저장된 뷰 기능 없음

### 노션 보완

- 직관적인 필터 UI
- 여러 조건 조합 가능
- 저장된 뷰로 자주 사용하는 필터 저장

### 구현 방법

#### 뷰 생성 예시

각 데이터베이스에 여러 뷰를 생성합니다:

**평가 데이터베이스**:
- "전체 평가" (기본 뷰)
- "진행 중인 평가" (상태 = In Progress)
- "이번 달 평가" (평가일시 필터)
- "미완료 평가" (상태 = Draft)
- "완료된 평가" (상태 = Confirmed)

**과제물 제출 데이터베이스**:
- "전체 제출물"
- "미제출 과제물" (제출일시가 비어있음)
- "최근 제출물" (제출일시 정렬)

**평가일정 데이터베이스**:
- "전체 일정"
- "진행 중인 일정" (상태 = In Progress)
- "예정된 일정" (상태 = Scheduled)

## 5. 자동화 및 워크플로우

### Supabase 단점

- 트리거와 함수 작성이 복잡
- 자동화 설정이 어려움
- 워크플로우 구성 제한적

### 노션 보완

- 수식 속성으로 자동 계산
- 롤업 속성으로 관계 데이터 집계
- 템플릿으로 일관된 데이터 생성

### 구현 방법

#### 수식 속성으로 자동 계산 필드 생성

**예시**: 평가 → 총점 (백분율)
```
if(prop("최대 점수 합계") > 0, round((prop("원점수 합계") / prop("최대 점수 합계")) * 100, 2), 0)
```

#### 템플릿으로 표준화된 레코드 생성

1. 평가 템플릿 생성
2. 기본값 설정:
   - 상태: Draft
   - 평가일시: 비어있음
3. 가이드 작성

**자세한 내용**: [롤업 및 수식 속성 활용](./06-rollups-formulas.md)

## 6. 모바일 접근성

### Supabase 단점

- 모바일 앱이 제한적
- 오프라인 접근 불가
- 푸시 알림 미지원

### 노션 보완

- 공식 모바일 앱 제공 (iOS, Android)
- 오프라인 접근 가능
- 푸시 알림 지원

### 활용 방법

1. **모바일 앱 설치**
   - iOS: App Store에서 "Notion" 검색
   - Android: Play Store에서 "Notion" 검색

2. **오프라인 접근**
   - 최근에 본 페이지는 오프라인에서도 접근 가능
   - 동기화는 온라인 상태에서 자동으로 수행

3. **푸시 알림 설정**
   - 설정 → 알림에서 알림 옵션 설정
   - @멘션, 댓글 등에 대한 알림 수신

---

## 7. 데이터 백업 및 복구

### Supabase 단점

- 백업 설정이 복잡함
- 복구 프로세스가 어려움

### 노션 보완

- 자동 백업 (클라우드 기반)
- 버전 히스토리 (페이지 수정 이력)
- 페이지 복구 기능

### 활용 방법

1. **버전 히스토리 확인**
   - 페이지 우측 상단 "..." 메뉴
   - "히스토리" 선택
   - 이전 버전 확인 및 복구

2. **페이지 복구**
   - 삭제된 페이지는 30일간 복구 가능
   - 휴지통에서 복구

---

## 8. 사용자 경험 (UX)

### Supabase 단점

- 기술적인 UI
- 초보자에게 어려움
- 학습 곡선이 가파름

### 노션 보완

- 직관적인 UI/UX
- 드래그 앤 드롭으로 쉬운 조작
- 다양한 블록 타입 지원

### 활용 방법

1. **드래그 앤 드롭**
   - 데이터베이스 레코드 순서 변경
   - 페이지 구조 재구성

2. **블록 활용**
   - 텍스트, 이미지, 비디오 등 다양한 콘텐츠
   - 데이터베이스 임베드
   - 코드 블록, 수식 등
