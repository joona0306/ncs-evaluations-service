# 대시보드 페이지 제작 가이드

## 개요

이 문서는 노션에서 NCS 평가 시스템의 대시보드 페이지를 제작하는 방법을 설명합니다. 와이어프레임과 단계별 가이드를 포함합니다.

## 대시보드 설계 원칙

### 1. 사용자 중심 설계

- 역할별 대시보드 구성 (관리자, 교사, 학생)
- 자주 사용하는 정보를 상단에 배치
- 직관적인 네비게이션

### 2. 시각화 우선

- 다양한 뷰 활용 (테이블, 보드, 타임라인, 갤러리)
- 색상 코딩으로 상태 표시
- 차트 및 통계 정보 표시

### 3. 실시간 정보

- 롤업 속성을 활용한 실시간 집계
- 필터를 통한 동적 데이터 표시
- 최신 정보 우선 표시

## 단계별 제작 가이드

### 1단계: 대시보드 페이지 생성

1. 노션에서 새 페이지 생성
2. 페이지 이름: `대시보드` (또는 역할별로 "관리자 대시보드", "교사 대시보드", "학생 대시보드")
3. 아이콘 및 커버 이미지 설정 (선택사항)

---

### 2단계: 통계 카드 추가

#### 방법 1: Callout 블록 사용

1. `/callout` 입력
2. 아이콘 선택 (📊, 📈 등)
3. 제목 입력: "전체 평가"
4. 내용 입력: 숫자 또는 수식

#### 방법 2: 데이터베이스 롤업 활용

1. 데이터베이스 임베드
2. 롤업 속성으로 통계 표시
3. 필요한 속성만 표시

#### 예시: 평가 수 통계

```
📊 전체 평가
[평가 데이터베이스의 Count 롤업]
```

---

### 3단계: 데이터베이스 임베드

#### 기본 임베드

1. `/database` 입력
2. 연결할 데이터베이스 선택
3. 뷰 선택 (테이블, 보드, 타임라인, 갤러리 등)

#### 뷰 설정

**테이블 뷰**:
- 속성 표시/숨김 설정
- 정렬 설정
- 필터 설정

**보드 뷰**:
- 그룹 기준 설정 (예: 상태)
- 정렬 설정
- 필터 설정

**타임라인 뷰**:
- 날짜 속성 설정
- 그룹 설정
- 필터 설정

**갤러리 뷰**:
- 이미지 속성 설정
- 정렬 설정
- 필터 설정

---

### 4단계: 필터 및 정렬 설정

#### 필터 설정

**예시: 진행 중인 평가만 표시**

1. 데이터베이스 뷰에서 "필터" 클릭
2. 조건 추가:
   - 속성: `상태`
   - 조건: `같음`
   - 값: `In Progress`
3. 저장

#### 정렬 설정

**예시: 최신순 정렬**

1. 데이터베이스 뷰에서 "정렬" 클릭
2. 정렬 기준:
   - 속성: `평가일시`
   - 순서: `내림차순`
3. 저장

---

### 5단계: 뷰 저장

자주 사용하는 필터와 정렬을 뷰로 저장합니다.

1. 뷰 설정 완료 후
2. 뷰 이름 변경 (예: "진행 중인 평가")
3. 자동으로 저장됨

## 역할별 대시보드 구성

### 관리자 대시보드

#### 섹션 1: 전체 현황

- 전체 평가 수
- 진행 중인 평가 수
- 완료된 평가 수
- 승인 대기 사용자 수

#### 섹션 2: 이번 주 평가 일정

- 평가일정 데이터베이스 (타임라인 뷰)
- 필터: 시작일시가 이번 주

#### 섹션 3: 평가 현황

- 평가 데이터베이스 (보드 뷰)
- 그룹: 상태
- 정렬: 평가일시 (최신순)

#### 섹션 4: 사용자 관리

- 프로필 데이터베이스 (테이블 뷰)
- 필터: 승인 상태 = false
- 정렬: 생성일 (최신순)

---

### 교사 대시보드

#### 섹션 1: 내 평가 현황

- 진행 중인 평가 수
- 완료된 평가 수
- 평균 점수

#### 섹션 2: 평가해야 할 항목

- 평가 데이터베이스 (보드 뷰)
- 필터: 교사 = [현재 사용자], 상태 = Draft
- 정렬: 생성일 (오래된 순)

#### 섹션 3: 내 평가 일정

- 평가일정 데이터베이스 (캘린더 뷰)
- 필터: 생성자 = [현재 사용자]

#### 섹션 4: 제출된 과제물

- 과제물 제출 데이터베이스 (갤러리 뷰)
- 필터: 평가일정의 생성자 = [현재 사용자]
- 정렬: 제출일시 (최신순)

---

### 학생 대시보드

#### 섹션 1: 내 평가 현황

- 진행 중인 평가 수
- 완료된 평가 수
- 평균 점수

#### 섹션 2: 제출해야 할 과제물

- 평가일정 데이터베이스 (보드 뷰)
- 필터: 종료일시 > 오늘, 상태 = In Progress
- 정렬: 종료일시 (가까운 순)

#### 섹션 3: 내 평가 목록

- 평가 데이터베이스 (테이블 뷰)
- 필터: 학생 = [현재 사용자]
- 정렬: 평가일시 (최신순)
- 표시 속성: 능력단위명, 총점, 상태

#### 섹션 4: 제출한 과제물

- 과제물 제출 데이터베이스 (갤러리 뷰)
- 필터: 학생 = [현재 사용자]
- 정렬: 제출일시 (최신순)

## 고급 기능

### 1. 동적 필터

노션의 필터 기능을 활용하여 동적으로 데이터를 필터링합니다.

**예시**: 날짜 범위 필터
- 시작일시: 이번 주
- 종료일시: 이번 달

### 2. 롤업 통계

관계된 데이터의 통계를 롤업 속성으로 표시합니다.

**예시**: 훈련과정별 평가 수
- 훈련과정 데이터베이스
- 평가 수 (Rollup → Count)

### 3. 수식 기반 표시

수식 속성을 활용하여 계산된 값을 표시합니다.

**예시**: 평균 점수
- 평가 데이터베이스
- 평균 점수 (Rollup → Average)

## 디자인 팁

### 1. 레이아웃

- 2열 또는 3열 레이아웃 사용
- 중요한 정보를 상단에 배치
- 섹션 구분 명확히

### 2. 색상 코딩

- 상태별 색상 설정 (Select 속성)
- 중요도에 따른 색상 사용

### 3. 아이콘 활용

- 섹션별 아이콘 사용
- Callout 블록에 아이콘 추가

---

## 문제 해결

### 문제 1: 데이터베이스가 너무 커서 느림

**해결**:
- 필터를 사용하여 필요한 데이터만 표시
- 페이지 크기 제한
- 불필요한 속성 숨김

### 문제 2: 롤업 속성이 업데이트되지 않음

**해결**:
- 페이지 새로고침
- 롤업 속성 재설정
- 관계 확인
