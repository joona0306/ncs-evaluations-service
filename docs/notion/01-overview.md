# 개요 및 시작하기

## 목표

이 가이드는 현재 Supabase 기반의 NCS(국가직무능력표준) 평가 시스템을 노션 데이터베이스로 구현하기 위한 종합 가이드입니다.

## 주요 목표

- ✅ Supabase의 관계형 데이터베이스 구조를 노션 데이터베이스로 변환
- ✅ 노션의 관계형, 롤업, 수식 속성을 활용한 데이터 모델링
- ✅ Supabase의 제약사항을 노션의 장점으로 보완
- ✅ 단계별 구현 가이드 제공

## 노션 데이터베이스 제약사항

노션 데이터베이스를 설계하기 전에 다음 제약사항을 이해해야 합니다:

### 1. 기본 속성 "이름"

- **변경 불가능**: 기본 속성인 "이름"은 변경하거나 삭제할 수 없습니다
- **활용 방법**: 가장 중요한 식별 정보로 사용해야 합니다
- **예시**: 프로필의 경우 "이름" = `full_name`, 훈련과정의 경우 "이름" = `name`

### 2. 속성 유형 변경

- **제한적**: 일부 속성은 생성 후 유형 변경이 불가능합니다
- **주의사항**: 처음부터 올바른 속성 유형을 선택해야 합니다
- **변경 가능**: Text ↔ Number, Date ↔ Created time 등 일부는 변경 가능

### 3. 관계형 속성

- **다른 데이터베이스 연결**: 다른 데이터베이스와 연결 가능
- **양방향 관계**: 양방향 관계 설정 시 자동으로 반대쪽 속성이 생성됨
- **제한사항**: 관계는 데이터베이스 간에만 설정 가능 (페이지와는 불가)

### 4. 롤업 속성

- **집계 계산**: 관계된 데이터의 집계 계산 가능
- **함수**: Count, Sum, Average, Min, Max, Unique values 등
- **제한사항**: 관계가 설정된 후에만 생성 가능

### 5. 수식 속성

- **계산 필드**: 다른 속성 기반 계산 가능
- **함수**: 다양한 수식 함수 제공 (if, round, dateBetween 등)
- **제한사항**: 복잡한 계산은 제한적일 수 있음

## 노션 데이터베이스의 장점

### 1. 시각화

- **다양한 뷰**: 테이블, 보드, 타임라인, 갤러리, 리스트, 캘린더 뷰
- **동적 필터**: 직관적인 필터 UI로 복잡한 쿼리 없이 데이터 필터링
- **정렬 및 그룹화**: 쉽게 데이터 정렬 및 그룹화

### 2. 협업

- **실시간 협업**: 여러 사용자가 동시에 작업 가능
- **댓글 및 멘션**: 레코드에 댓글 추가 및 @멘션으로 알림
- **권한 관리**: 세밀한 권한 설정 가능

### 3. 자동화

- **수식 속성**: 자동 계산 필드
- **롤업 속성**: 관계 데이터 자동 집계
- **템플릿**: 일관된 데이터 입력을 위한 템플릿

### 4. 모바일 접근성

- **공식 앱**: iOS, Android 앱 제공
- **오프라인 접근**: 오프라인에서도 일부 기능 사용 가능
- **푸시 알림**: 알림 지원
